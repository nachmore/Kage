# Kage Code Review — July 2026

Full multi-dimensional review across architecture, correctness, security, performance, frontend/UX, the extensions repo, and system integration. Mechanical gates (fmt, clippy, JS tests) were run alongside targeted agent reviews; every finding below was verified against the actual code, not just pattern-matched.

**Gate status at review time:** `cargo fmt --check` ✅ · `cargo clippy --all-targets` ✅ (clean) · JS tests 876/876 ✅ · `check_i18n.py` ❌ (93 errors — spotify extension locale drift) · `npm audit` ⚠️ (dev-only: vitest critical, uuid moderate).

Severity legend: **P0** ship-blocker / security · **P1** high-impact bug or perf · **P2** worth doing · **P3** polish.

---

## Security (extension trust boundary)

The sandbox itself is well built — zip-slip/symlink defense, extension-id validation, the HTML *fragment* sanitizer, Rust-authoritative capability normalization, MessagePort isolation, and fail-safe updater signature checks are all solid. The holes are in the **capability→command mapping**, not the sandbox.

- **P0 — `get_config` leaks the entire app config to any `storage` extension.** `ui/js/shared/extension-permissions.js:24` maps `get_config` to `storage` with the comment "scrubbed in host" — but no scrubber exists. `src/commands/system/config_io.rs:19` returns `config.clone()`. The `storage` capability is marketed as the *safest* and is the implicit default for permission-less manifests, yet it exposes `telemetry.install_id` (the ID the design elsewhere hides), every other extension's stored config and grants, ACP connection commands/URLs, context rules, hotkeys, and tool-permission policy. Fix: remove `get_config` from the extension-reachable map, or add a real host-side whitelist scrubber.
- **P0 — Cross-extension config read/write via unscoped `id`.** `get_extension_config` / `save_extension_config` are gated by `storage` but are NOT in the host's `STORAGE_COMMANDS` identity-injection set (`extension-sandbox-host.js:74`), so their `id` param passes untrusted from sandbox to Rust with no ownership check. Extension A can read/overwrite extension B's config. Breaks the stated per-extension isolation guarantee. Fix: inject the authoritative extension id host-side, or drop the `id` param and resolve it from the calling sandbox.
- **P2 — `filesystem` capability silently includes destructive ops.** Described to users as "Scan folders and search files" but also gates `execute_folder_plan` (move/rename/delete) with a caller-supplied `root`. Fix: split destructive ops into their own capability or disclose in the description.
- **P3 — Reflected input in OAuth loopback page.** `src/commands/oauth.rs:303` interpolates `success_label` and the `error` query param into served HTML without escaping. Low blast radius (single-shot ephemeral localhost port). Fix: HTML-escape both.
- **P3 — Shared frecency file writable by any `storage` extension** (`config_io.rs:103`); **best-effort package integrity** — SHA-256 only checked when the catalog includes it (`store.rs:624`).

## Correctness / bugs (Rust)

- **P1 — 60s hard timeout kills healthy long agent turns.** `src/acp_client/transport.rs:449` applies a flat 60s `recv_timeout` to every request, but a `session/prompt` response only arrives when the turn ends. Any turn with >60s of generation/tool use hits the recovery ladder → `restart_connection()` kills the agent mid-turn and re-sends. The `last_activity` field that could fix this already exists but is never read (`transport.rs:64`, dead). Fix: activity-aware deadline for `session/prompt`.
- **P1 — UTF-8 byte-slice panic in `invoke_subagent`.** `src/acp_client/session.rs:464`: `&query[..query.len().min(100)]` slices at byte 100 with no char-boundary check. Any non-ASCII sub-agent query (31 locales ship) with a multibyte char straddling byte 100 panics. Fix: `query.chars().take(100).collect()`.
- **P1 — Orphan-cleanup name match is dead code.** `src/process_manager.rs:67`: `name_lower.contains("Kage")` — a lowercased string can never contain a capital "Kage", and there's no "kiro" pattern. Only `node`/`npx` wrappers ever get reaped; on macOS/Linux (no Job Object) orphaned agent backends survive every restart. Fix: compare lowercase literals + include real agent binary names.
- **P1 — Single global `pending_permission` slot vs. multi-session.** `src/state.rs:24` is one `Option`. Two overlapping `session/request_permission`s (two windows/sessions) overwrite the slot; the first stays open agent-side forever, wedging that session. Fix: key pending permissions by request id.
- **P2 — Naive `split_whitespace` spawn parsing.** `transport.rs:236` breaks any `spawn_command` whose program path contains spaces (`C:\Program Files\...`). Fix: shell-words tokenizer or store program+args separately.
- **P2 — `ProcessManager::terminate` force-kill branch is unreachable.** `process_manager.rs:160` — the `join()` blocks indefinitely; the timeout/force-kill fallback is dead. If `child.kill()` fails, `terminate()` hangs (called from `Drop`, signal handlers, `disconnect`). Fix: `try_wait` poll loop with a real deadline.
- **P2 — Chat window "response ready" notification can never fire.** `ui/js/chat/app.js:291` sets `isWaitingForResponse=false`, then line 326 gates the notification on `isWaitingForResponse && !focused` — always false. Users never get chat completion notifications (floating works). Fix: drop the `isWaitingForResponse` term.

## Convention / correctness (privacy, i18n, config)

- **P1 — Full user prompt logged at `info!`.** `src/commands/messaging/streaming.rs:18` writes every prompt verbatim to `app.jsonl`; the ACP layer deliberately gates this behind `debug_mode`. Users share app.jsonl in bug reports. Fix: log length only.
- **P2 — Missing `#[serde(default)]` on top-level config fields.** `Config.hotkey/.acp/.ui/.system`, `UiConfig.floating_window_opacity`, `SystemConfig.auto_start` (`src/config.rs:13`). A partially-written config missing any of these triggers the full backup-and-reset path, losing grants/hotkeys.
- **P2 — Unthrottled `config.save()` on every extension tool call** (`permissions.rs`, to bump `last_seen`) does an fsync under the global config mutex on a tokio worker — contrast the carefully throttled+off-thread save the codebase uses elsewhere.
- **P2 — Idle silent-install ignores `chat-<uuid>` windows.** `src/updater.rs:144` `USER_WINDOW_LABELS` omits secondary chat windows; a user working only in one can be auto-updated out from under mid-conversation. Fix: enumerate `app.webview_windows()`.
- **P3 — User-facing English bypasses i18n** in several Rust paths (`streaming.rs:43,110`, `updater.rs` changelog/install errors, `updates.rs`); and timestamp-injection + rate-limit error state are process-global rather than per-session (`session.rs:38`, `notifications.rs:338`).

## Performance

- **P1 — Streaming markdown re-parses the whole message every 150ms tick.** `ui/js/shared/markdown.js` freezes a stable prefix, but when the tail is one long fenced code block or paragraph (the common case for streamed code), the split point stays at 0 and `marked.parse()` runs over the entire growing buffer each tick → ~O(n²). A 50KB response ≈ 130 full re-parses + full `innerHTML` replacement + Prism re-highlight. Also, when the prefix *does* advance it rebuilds the frozen DOM via `innerHTML` rather than appending the delta. Fix: parse only the delta and append; cap/freeze inside long fences.
- **P1 — Per-chunk O(n) scans of the full accumulator.** `message-stream-controller.js:88` runs `detectAutomationPlan` (full regex, no `includes()` pre-check), two `indexOf` scans, and — when TTS is on — `splitSentences`+`cleanForTts` (30+ `replaceAll` passes + Unicode emoji regexes) over the *entire* accumulated text on every 16ms chunk. Fix: cheap `includes()` gate; track a processed-offset and only scan the unprocessed suffix.
- **P1 — UIA tree walk does ~5,000 cross-process COM calls per snapshot.** `src/os/windows/accessibility.rs`: 500 elements × ~10 separate property/pattern COM round-trips each (`get_states`/`get_actions` probe ~5 patterns apiece). This is why an 8s timeout is needed. Fix: `IUIAutomationCacheRequest` to batch per subtree — typically 5–10× faster.
- **P1 — App icons (base64 PNGs) shipped over IPC on every keystroke.** `src/commands/input.rs` + `app_launcher.rs`: `find_app` clones the full `Application` (icon String included) for every match, then embeds up to 5 icons (~50–150KB) in each debounced-keystroke response, though the JS diffing renderer reuses nodes anyway. Fix: score `&app` refs before cloning; send icon *keys* and let JS cache/fetch.
- **P2 — `config.save()` on the floating-hide hotkey path.** `src/commands/window.rs:233` does a synchronous full-config serialize+fsync before `window.hide()` returns when `window_start_position=="remember"`. Fix: update in-memory + hide immediately + `spawn_blocking` the save.
- **P2 — Floating window resize+IPC storm during streaming.** `ui/js/floating/window.js`: each 150ms render fires ResizeObserver → forced layout + `currentMonitor()` + `resize_floating_window` + `_ensureOnScreen` (3–4 more IPC), ~6 IPC/150ms even when height didn't change. Fix: hoist the cheap short-circuit before the monitor IPC; cache the monitor cap.
- **P2 — Unbounded growth:** per-session streaming accumulators live for app lifetime, only cleared on the session's *next* prompt (`acp_client/mod.rs:195`) — evict on `message_complete`; `activity.db` has no retention (one row/5s forever) and `build_report` loads every row into a `Vec` (`activity_tracker.rs`) — add a retention cutoff + SQL aggregation; chat DOM never virtualized (`chat/app.js:1547`) — window to last ~50 or `content-visibility:auto`.
- **P3 — Chunk-flush thread wakes 60×/s for the whole app lifetime even when idle** (`notifications.rs:374`); `_processAppIcons` serializes the whole message DOM to a string per render tick (`markdown.js:187`); smooth-scroll restarted every chunk (`chat/app.js:2808`).

**Confirmed solid (no action):** the Rust→JS chunk batcher, `emit_filter` audience scoping, search `_searchGeneration` cancellation guard, config-cache memoization, DOM-diffing search renderer, and deferred startup init are all well done.

## Frontend XSS / UX (privileged webview)

The chat/floating webviews have `window.__TAURI__`, so unescaped injection here is real. `hardenMarkedOnce` and the widget circuit-breaker are exemplary; these are the gaps:

- **P1 — Unescaped `error.message` into `innerHTML`.** `ui/js/shared/markdown.js:1021,1060`: mermaid/graphviz errors echo the (agent-controlled) diagram source; a malformed fence with `<img onerror=...>` executes. Fix: `textContent`.
- **P1 — Slash-command options injected unescaped.** `ui/js/floating/app.js:1093`: agent `commands/execute` label/description raw into `innerHTML`; the chat window's equivalent (`chat/app.js:1009`) correctly uses `textContent`. Same for shortcut/app names in `floating/suggestions.js:29,63,162` (the newer `search-unified.js` escapes; these legacy renderers don't). Fix: escape / consolidate onto the escaped renderer.
- **P2 — mermaid `securityLevel:'loose'`** (`markdown.js:99`) allows click callbacks / HTML labels in agent diagrams; and an inline `onerror` attribute interpolates a label char (`chat/app.js:2682`, `suggestions.js:56,153`) — a label starting with `'` breaks out. Fix: `'strict'`; real `onerror` handler on a created element.
- **P2 — Reuse:** substantial near-identical logic across `chat/app.js` and `floating/app.js` (global keyboard shortcuts, tab-cycle completion, image→base64, copy-button SVG swap, toolbar host effects) belongs in `ui/js/shared/`. Two divergent `formatRelativeDate` and four duplicate `escapeHtml` copies exist.
- **P2 — UX parity/consistency:** floating `showError` has no reconnect affordance (chat does); chat input is `disabled` during streaming (floating lets you type); native `confirm()`/`alert()` used in ~10 places despite a themed `confirm-dialog.js`; session sidebar rows aren't keyboard-accessible (no tabindex/role/keydown).
- **P2 — `showStatus` 5s auto-hide timer** (`settings/manager.js:511`) is never cancelled and can hide the persistent restart-prompt mid-read.
- **P3 — Hardcoded English strings bypassing `t()`** across floating Ollama widget, chat prompt notes, sidebar labels, agent-sessions, and store empty-states (enumerated in the frontend findings); string-concatenated errors in `speech.js`/`shortcuts.js`; hardcoded error-red hexes (`#c44`/`#dc2626`/…) — introduce `--kage-error`/`--kage-danger` tokens; sole `sessionStorage` use (`floating/app.js:2141`) should be in-memory or documented.

## Extensions repo (`../Kage-Extensions`)

Healthy overall: 278/278 tests pass, all 19 manifests validate, no secrets, no plaintext http. Top issues:

- **P1 — spotify locale drift blocks host ingest.** 3 keys (`help.commands_markdown`, `settings.commands.html`, `settings.commands.label`, added in commit 2dce1a5) missing from all 31 non-EN catalogs → 93 errors from Kage's `check_i18n.py`. Fix: run `scripts/translate.py`.
- **P1 — Repo's own i18n gate can't detect key-set drift.** `scripts/check-i18n.mjs` only flags empty/TODO messages, which is why #1 passed local CI. Fix: add an EN-vs-non-EN key-set diff.
- **P1 — `check:versions` failing (CI red).** snippets, spotify, todos, window-walker (and per the other pass, bookmarks/calendar/dictionary/focus-tracker/hello-world) changed source without a manifest version bump → clients never pull the fixes. Fix: bump versions.
- **P2 — link-preview hero `og:image` src unvalidated** (`formatter.js:162`) while the favicon path right below checks `/^https?:/i`. Fix: apply the same scheme check.
- **P2 — dictionary reaches across the sandbox** importing host-internal `js/shared/language-detect.js` by URL guessing; caches transient 5xx as "not_found". github declares a dead `personal_token` config field (token actually in extension-data) and has no rate-limit UX. weather sets a forbidden `User-Agent` header (silently dropped) and matches trigger without a word boundary (`weatherx` geocodes `x`).
- **P2 — No fetch timeout** on weather/currency/github/dictionary network calls; only spotify has `fetchWithTimeout` (with a comment documenting the 5–32s hang it prevents). Promote it to a shared helper. Similar duplication: `escapeHtml` (4–5 copies) and the extension-data cache read/write pair.
- **P3 — Missing `getKeywords()`** on ~9 trigger-driven extensions; currency ships stale `HRK`; several manifests omit `tags`; only spotify has a README; hello-world reference uses unbound `context.i18n.t`.

## System integration & docs

- **P2 — Cross-platform coverage is uneven and some paths degrade silently.** Linux `accessibility.rs`, `hotkey.rs`, `icon.rs` are explicit stubs; macOS has an `ax_worker.rs` but Linux/macOS cover only common paths. Verify each stub returns a *clear* error rather than an empty success (Linux hotkey capture just `warn!`s). No Job-Object equivalent exists on macOS/Linux, which ties into the dead orphan-cleanup match (P1 above) — orphaned backends can accumulate.
- **Docs are broad and mostly healthy** (COMPUTER_CONTROL_MCP, EXTENSIONS, I18N, OS_ABSTRACTION_GUIDE, SECURITY_MODEL, RELEASE, PRIVACY, TOOL_PERMISSIONS, SHORTCUTS_GUIDE, DEBUG_MODE all present). CLAUDE.md's specific claims that were spot-checked (state-on-Builder, os abstraction, no `Result<T,String>` commands) all held. Gaps worth filling: the `get_config` "scrubbed in host" comment is actively false (see P0); SECURITY_MODEL should document the `storage`-capability breadth once fixed.
